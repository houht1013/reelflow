# Reelflow MVP 产品与架构方案

> 版本：v0.1  
> 状态：MVP 范围确认稿  
> 日期：2026-06-19  
> 基础工程：TinyShip SaaS Template  
> 主技术栈：TanStack Start + TypeScript + Drizzle + Postgres + 独立 TypeScript Worker  

---

## 1. 产品定位

Reelflow 是面向短视频内容生产的参数化模板工作流 SaaS。

第一阶段不做完整在线剪辑器，也不追求一键 100% 完美成片。产品目标是自动完成 90% 的短视频生产工作，剩余 10% 通过人工审核、AI 补图/补语音、剪映二次编辑完成。

一句话定位：

> 面向小白和内容运营者的爆款短视频模板生成 SaaS：选择模板、填写参数，即可生成可编辑剪映草稿包，并可选生成 1080P MP4。

核心价值：

- 小白也能用官方模板开始起号。
- 内容运营者可以稳定复用爆款结构。
- 官方可承接私有模板定制服务。
- 生成结果既可以作为剪映草稿二次编辑，也可以云端渲染为 MP4。

前端体验原则：

- 在功能实现的前提下，必须追求最佳可用性和交互体验。
- 用户默认按计算机小白理解，不暴露代码、节点、工作流、时间线等实现复杂度。
- 关键流程必须清晰：选模板、填参数、预估积分、提交任务、查看进度、预览产物、下载结果。
- PC 和移动端都需要适配，移动端不应只是桌面页面压缩。
- 阶段追溯用于让用户心里有底，不用于让用户调试系统。
- 管理端可以复杂，用户端必须克制。

## 2. MVP 用户范围

MVP 同时覆盖两类用户，但不拆成两套产品：

1. 小白个人用户
   - 想探索短视频赛道。
   - 不懂剪辑、不懂工作流。
   - 需要官方模板直接跑起来。

2. 矩阵运营者 / 内容工作室
   - 有持续内容生产需求。
   - 关注模板复用、效率、成本、私有模板。
   - 后续可能需要批量、定时、团队协作。

MVP 的体验应优先服务小白：模板少、参数少、流程清楚、交付明确。

## 4. 技术栈决策

### 4.1 主应用

只使用 `apps/tanstack-app` 作为 Reelflow MVP 主应用。

不再要求 Next / Nuxt 同步实现。当前 TinyShip 中的 Next 和 Nuxt 可以保留在仓库中，但不作为 Reelflow MVP 的交付目标。

### 4.2 运行时

MVP 使用独立 TypeScript Worker：

```text
apps/tanstack-app          # 用户端 + 管理端
apps/execution-worker      # 任务运行时
libs/reelflow              # Reelflow 共享领域逻辑
libs/database              # Drizzle schema 和数据库访问
config/reelflow.ts         # Reelflow 配置
```

暂不引入 Python。未来如果出现复杂图片、音频、视频本地处理，再单独拆出 Python media service。

### 4.3 数据库

数据库使用 PostgreSQL。Reelflow 新业务 schema、迁移和运行时逻辑不考虑 SQLite/D1 兼容。

Drizzle 继续作为数据库 schema 和迁移主源。Worker 与 Web 共享同一数据库，通过表结构和状态机作为契约。

TinyShip 是基础脚手架，不是不可改的外部依赖。现有用户、订单、订阅、积分、存储、AI provider 等能力可以按 Reelflow 业务需要复用、合并或改造。MVP 实施上优先渐进改造，避免一次性重写带来不必要风险。

### 5.1 业务管理端

- 模板库
- 生成页

- AI 语音
- 订阅 / 积分
- 邀请奖励
- 通知中心

管理后台包括：

- 模板管理
- 私有模板授权
- 价格清单
- Provider 健康和启停
- 任务查看和重试
- 用户 / workspace 管理
- 订单 / 订阅 / 积分流水
- 邀请记录
- 运营指标

### 5.2 运行时

由 `apps/execution-worker` 承载：

- 任务领取

- 调度优先级
- 阶段执行

- 实际结算

- 通知触发

运行时 MVP 不做复杂 worker dashboard，不暴露复杂 RPC。提供进程入口和必要健康检查即可。

## 6. 模板体系

### 6.1 模板类型

MVP 首发 3 个官方模板：

1. 心理学火柴人
2. 认知 / 观点口播
3. 清单知识卡片

模板方向聚焦：

- 知识分享
- 观点表达
- 情绪价值
- 认知提升
- 故事化口播

短剧 / 漫剧原生生产不进入 MVP。后续可以作为独立生产模式设计。

### 6.2 模板实现

模板采用高代码注册方式：

- Builder 由官方开发。
- 用户不编辑模板结构。
- 用户不接触代码、节点、工作流、时间线。
- 管理后台只管理模板元数据、上下架、推荐、授权、参数 schema 展示。

私有模板由官方承接开发，并授权给指定 workspace 使用。

### 6.3 推荐模板

推荐模板是运营配置，不写死在产品逻辑中。

模板支持：

- `recommended`
- `featured_order`
- `landing_position`
- 公开 / 私有
- 指定 workspace 授权

- 单行文本

- 开关

- 图片上传 / 选择资产
- 音色选择
- 语言选择
- 隐藏默认参数

字段能力：

- 必填 / 选填

## 8. 任务系统

### 8.1 任务模式

MVP 只支持单次手动生成。

暂不支持：

- CSV 批量
- 预约执行
- 周期执行
- 社媒发布

任务是异步长任务，由 execution worker 执行。

### 8.2 任务状态

任务执行状态：

- `queued`
- `running`
- `completed`
- `failed`

质量状态：

- `unchecked`
- `accepted`
- `needs_fix`

质量问题类型：

- `missing_image`
- `bad_image`
- `missing_audio`
- `bad_audio`
- `caption_issue`
- `draft_issue`
- `render_issue`

任务完成不等于 100% 完美。只要主要交付物可生成，缺图、少音频、局部素材不合格属于质量问题，不作为任务失败。

### 8.3 阶段追溯

用户侧展示：

- 总进度条
- 当前阶段
- 阶段状态
- 阶段产物预览
- 预估积分 / 冻结积分 / 实际消耗
- 质量问题提示
- 下载入口
- 重新生成
- 失败重试

后台展示：

- 每阶段开始 / 结束时间
- provider / model / plugin
- 输入输出摘要
- token / 次数 / 时长 / 图片数
- cache hit
- 错误码和错误日志
- worker id
- attempt id
- 重试记录

统一阶段建议：

- `precheck`
- `script`
- `storyboard`
- `image`
- `voice`
- `caption`
- `compose_project`
- `draft_package`
- `render_mp4`
- `settlement`
- `notify`

### 8.4 失败和重试

MVP 支持：

- 完整重新生成
- 失败点重试

局部补图、补语音不在任务详情中做复杂局部编辑，走独立 AI 工具区。

失败点重试规则：

- 已完成阶段复用
- 已发生成本保留
- 只对新增阶段继续计费
- 生成 retry attempt 记录

## 9. 预检和健康检查

任务执行前必须预检：

- 模板是否可用
- workspace 订阅 / 积分状态
- 预估积分是否可冻结
- 并发额度是否允许
- 所需 provider 是否启用
- LLM / TTS / 生图 / 草稿服务是否可用
- 用户上传资产是否存在
- 官方模板依赖资产是否存在
- `capcut-mate` 服务是否可访问
- 选择云端 MP4 时，云渲染服务是否可用

预检失败时，任务不进入执行。

管理后台需要最小 provider 健康检查：

- 查看 provider 状态
- 查看最近错误
- 手动触发健康检查
- 启用 / 停用 provider
- 前台提交任务时避开停用 provider

---

## 10. 调度和资源控制

MVP 需要任务锁、幂等、断点续跑。

运行时必须支持：

- 任务锁
- 锁超时恢复
- `attempt_count`
- `locked_by`
- `locked_at`
- 阶段状态落库
- workspace 并发限制
- workspace 队列长度限制

调度策略：

- 同一 workspace 内 FIFO
- 全局按套餐优先级 + 提交时间排序
- 付费用户优先于免费用户
- 高级套餐优先于基础套餐
- 已 running 任务不抢占
- 失败重试可略高优先级
- 后台可手动提权

未来大客户可以单独分配独立 worker 池或专属通道。

## 11. 交付物

### 11.1 剪映草稿包

剪映草稿包是 MVP 核心交付物。

参考开源项目：

- `Hommy-master/capcut-mate`

MVP 必须支持：

- 生成草稿包
- 下载草稿包
- 使用本地转换工具让剪映专业版打开

第一版优先支持 Windows。

### 11.2 云端 MP4

云端 MP4 是可选交付物，不替代剪映草稿包。

规则：

- 固定 MP4
- 固定 1080P
- 不提供分辨率、码率、帧率等用户配置
- 通过外部云渲染 API provider 执行
- 异步执行
- 渲染失败不影响草稿包交付
- 产物进入资产库
- 按实际成本和对外积分价扣费

### 11.3 社媒发布

MVP 不做社媒发布。

数据结构可预留 `publish_targets`，但 UI 和业务流程不开放。

## 12. 资产库

统一术语为资产库，不再称素材库。

资产分两类：

1. 任务产物资产
   - 文案
   - 分镜
   - 图片
   - 音频
   - 字幕
   - 剪映草稿包
   - MP4
   - 执行日志摘要

2. 用户上传资产
   - logo
   - 头像
   - 产品图
   - 背景图
   - 参考图
   - 后续扩展音频 / BGM

资产模型需要支持未来：

- 从已有图片生成新图
- 从已有视频生成变体
- 复用历史文案 / 分镜
- 选择历史任务产物作为新任务输入
- 资产归属 workspace
- 资产和任务、模板、参数之间可追溯

MVP UI 可以简单，但数据模型不要做窄。

## 13. 独立 AI 工具区

MVP 包含：

- AI 生图
- AI 语音

暂不包含：

- AI 视频

AI 工具产物进入资产库，可供后续模板任务复用。

使用场景：

- 补图
- 替换不合格图片
- 补旁白
- 替换不满意配音

计费规则：

- 生图按模型 / 张数 / 尺寸等计费
- 语音按字符数或时长计费
- 失败按实际供应商成本规则处理

## 14. Workspace 模型

底层使用 `workspace`，前台 MVP 暂不暴露该概念。

`workspace` 是资源、计费、权限归属容器，不等同于团队。

关系：

```text
User
  -> WorkspaceMember
  -> Workspace
       -> Subscription
       -> CreditAccount
       -> Assets
       -> Jobs
       -> PrivateTemplates
```

MVP 表现：

- 注册后自动创建默认 workspace。
- 用户看到的是“我的任务”“我的资产”“我的积分”。
- 订阅、积分、任务、资产、私有模板都归属 workspace。

未来团队协作通过 workspace 成员和角色扩展。

## 15. 订阅、积分和计费

### 15.1 商业模型

MVP 支持：

- 免费试用积分
- 月度 / 年度订阅套餐
- 订阅发放积分
- 额外积分包
- 定制模板线下承接

暂不做：

- 模板在线单独购买
- 企业报价
- 优惠券
- 发票系统
- 自动退款流程

### 15.2 积分归属

订阅和积分归属 workspace。

现有 TinyShip 的 user 级积分需要改造为 workspace 级积分账户。

### 15.3 扣费策略

任务提交时：

- 估算积分
- 冻结预估额度
- 通过预检后入队

执行中：

- 记录真实用量
- 允许小额超额继续执行

完成后：

- 计算实际消耗
- 余额充足则结算并开放下载
- 余额不足则任务完成但产物锁定
- 用户补充积分后自动解锁

### 15.4 价格清单

能力价格清单需要记录：

- provider
- model / plugin
- 计费单位
- 供应商成本价
- 对外积分价
- 是否启用

不做价格版本表。

每条 usage / ledger 记录冗余当时价格快照：

- provider
- model / plugin
- usage amount
- unit
- provider cost
- credit cost
- pricing snapshot

后续调价通过站内通知和邮件告知，并保留调价备忘记录。

## 16. 免费试用

MVP 免费策略：

- 注册赠送试用积分
- 试用积分可跑官方模板
- 高成本能力限制
- 免费用户并发低
- 免费用户队列优先级低
- 免费用户资产保留期可更短

不添加 Reelflow 平台水印。

用户账号水印仍可作为模板参数存在。

## 17. 邀请奖励

MVP 不做完整分销。

MVP 做：

- 邀请链接 / 邀请码
- 注册归因记录
- 邀请成功赠送积分
- 被邀请人注册奖励积分
- 后台保留邀请记录

暂不做：

- 分销中心
- 佣金台账
- 人工结算
- 自动提现
- 多级分销

后续可升级为会员分销体系。

## 18. 通知

MVP 通知渠道：

- 站内通知
- 邮件通知

通知事件：

- 任务完成
- 任务失败
- 任务存在需补齐项
- 任务产物因欠费锁定
- 充值 / 订阅支付成功
- 积分余额不足
- 私有模板开通
- 邀请奖励到账
- 资产 / 产物即将过期

暂不做短信、微信、飞书、Telegram。

## 19. 内容安全

MVP 先做规则模式安全围栏。

重点规避政治敏感红线和明显违规内容。

检查点：

- 用户输入参数
- 模板 prompt 拼接结果
- LLM 输出文案
- 生图 prompt
- AI 语音文本

命中高风险规则：

- 阻止执行或阻止该阶段生成

命中低风险规则：

- 记录风险标签
- 后台可见

后续预留接入第三方或模型审核服务。

---

## 20. 保留策略

MVP 保留策略：

- 任务详情长期保留
- 计费流水长期保留
- 草稿包 / MP4 默认保留 30 天
- 图片 / 音频 / 字幕 / 文案默认保留 90 天
- 后台详细执行日志保留 30 天
- 用户保存到资产库的资产长期保留，受套餐容量限制
- 免费用户可配置更短保留期
- 到期前站内通知 / 邮件提醒

保留期先做系统配置，不做复杂套餐差异。

## 22. TinyShip 现有能力复用与改造

### 22.1 可复用能力

- TanStack Start 应用
- Better Auth 认证

## 23. 部署形态

本地开发：

```text
pnpm dev:tanstack
pnpm dev:worker
```

生产部署：

```text
reelflow-web       # TanStack Start 用户端 + 管理端
reelflow-worker    # Execution Worker
postgres
object-storage
capcut-mate service / desktop client chain
cloud-render provider
```

部署原则：

- Web 和 Worker 独立部署。
- Worker 可水平扩容。
- Worker 崩溃不影响登录、支付、查看任务。
- Web 发版不应影响 running 任务。
- Cloudflare 可作为研发期和生产期的主要服务方；需要真实资源验证时，可以直接创建和配置对应资源。
- 后续大客户可分配独立 worker 池。
